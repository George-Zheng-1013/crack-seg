## 1. 项目概述

本项目构建了一套面向基础设施外观缺陷的智能检测系统。系统以计算机视觉为核心，能够对裂缝、破损/剥落、蜂窝麻面、孔洞、钢筋外露、渗水六类常见混凝土表面缺陷进行自动识别、定位、可视化标注和报告导出。

技术路线采用 YOLOv26-Seg 分割模型作为主检测引擎，结合 SigLIP 视觉语言模型进行缺陷成因语义分析，最终通过几何特征与语义匹配的融合决策输出检测结果。系统以 FastAPI 为后端、单页 HTML 为前端，支持图像、批量图像和视频三种输入模式。

## 2. 数据工程

### 2.1 数据来源

项目使用的数据集由三个来源合并而成：

| 数据集    | 来源     | 原始类别                                                 | 图片数量（train/val/test） |
| --------- | -------- | -------------------------------------------------------- | -------------------------- |
| crack-seg | 老师提供 | 1类：crack                                               | 3717 / 112 / 200           |
| CUBIT-Det | 老师提供 | 3类：Crack, Spalling, Moisture                           | 304 / 35 / 82              |
| GYU-DET   | 自行搜集 | 6类：Crack, Breakage, Comb, Hole, Reinforcement, Seepage | 7272 / 966 / 963           |

### 2.2 数据合并流程

三个数据集的原始标注格式不同（crack-seg 为分割标注，CUBIT-Det 和 GYU-DET 为检测框标注），需要统一处理后才能用于分割模型训练。合并流程如下：

1. **标注格式统一**：将所有数据集转换为 YOLO 分割格式（class_id + 归一化多边形坐标）。
2. **SAM 模型生成 mask**：对 CUBIT-Det 和 GYU-DET 中仅有检测框标注的图像，使用 Segment Anything Model（SAM）根据边界框提示自动生成像素级分割 mask。
3. **类别映射**：将 CUBIT-Det 的 Spalling 映射为 Breakage，Moisture 映射为 Seepage；crack-seg 的 crack 直接对应 Crack 类别。
4. **数据合并**：三个来源的数据按 train/valid/test 划分合并为统一数据集。

合并工具代码位于 `crack-seg/算法/` 目录。

### 2.3 最终数据集：crack-multi-seg

| -------------- | --------------------------------------------------- |

### 2.4 数据增强

训练过程中采用的数据增强策略包括：

- **Mosaic**（概率 1.0）：将 4 张图像拼接为一张，增强小目标和多尺度检测能力。
- **水平翻转**（概率 0.5）：随机水平镜像。
- **缩放**（范围 0.5）：随机缩放图像。
- **随机擦除**（概率 0.4）：随机遮挡部分区域，提升模型对遮挡的鲁棒性。
- **HSV 色彩抖动**：色调偏移 0.015，饱和度变化 0.7，明度变化 0.4。
- **仿射变换**：平移 0.1，无旋转和剪切。

Mosaic 增强在训练最后 10 个 epoch 关闭（`close_mosaic: 10`），让模型在后期专注于精细化学习。

## 3. 算法选型与依据

### 3.1 检测框架选型

项目评估了三种主流方案：

| 方案 | 框架        | 优点                                                           | 缺点                                | 结论           |
| ---- | ----------- | -------------------------------------------------------------- | ----------------------------------- | -------------- |
| A    | YOLOv26-Seg | 推理速度快，同时输出 mask、bbox 和置信度，Ultralytics 生态完善 | 细小裂缝边缘精度可能略低            | **选用** |
| B    | U-Net       | 像素级精细分割，医学/工业领域验证充分                          | 需额外后处理生成 bbox，推理速度较慢 | 备选           |
| C    | SegFormer   | Transformer 全局建模能力强                                     | 参数量大，显存占用高，硬件门槛高    | 不采用         |

选择 YOLOv26-Seg 的核心理由：

1. **输出格式兼容**：原生输出 mask 多边形、边界框和置信度，与系统的可视化和报告模块天然匹配，无需额外后处理。
2. **推理速度快**：单张 640×640 图像在 GPU 环境下推理时间约 185ms，满足交互需求。
3. **工程部署便捷**：Ultralytics 提供统一的 Python API，训练、验证、导出、推理全流程代码量小。

### 3.2 具体模型版本

训练使用的基础模型为 `yolo26m-seg.pt`（YOLO26-Medium 分割版），这是一个中等规模的分割模型，在精度和速度之间取得较好平衡。模型参数量为 23.6M，FLOPs 为 121.5B，适合在消费级 GPU 上训练。

**YOLO26-Seg 系列模型对比**：

| 模型 | 输入尺寸 | mAPbox 50-95 | mAPmask 50-95 | 参数量 (M) | FLOPs (B) |
|------|----------|--------------|---------------|------------|-----------|
| YOLO26n-seg | 640 | 39.6 | 33.9 | 2.7 | 9.1 |
| YOLO26s-seg | 640 | 47.3 | 40.0 | 10.4 | 34.2 |
| **YOLO26m-seg** | **640** | **52.5** | **44.1** | **23.6** | **121.5** |
| YOLO26l-seg | 640 | 54.4 | 45.5 | 28.0 | 139.8 |
| YOLO26x-seg | 640 | 56.5 | 47.0 | 62.8 | 313.5 |

选择 m 版本在精度和训练成本之间取得平衡。

### 3.3 类别加权损失函数

针对数据集中类别不平衡的问题（Hole、Seepage 类样本数量较少），项目实现了自定义的加权分割损失函数。

**设计思路**：

```python
def make_class_weights(counts, max_weight=3.0):
    """使用 sqrt 逆频率计算类别权重，设置上限避免过度关注少数类"""
    safe_counts = np.maximum(counts, 1)
    weights = np.sqrt(safe_counts.max() / safe_counts)
    weights = np.clip(weights, 1.0, max_weight)
    return torch.tensor(weights, dtype=torch.float32)
```

**关键设计**：

- 使用 `sqrt` 而非直接逆频率，避免少数类权重过大导致模型对多数类精度下降
- 设置权重上限 3.0，平衡各类别的贡献
- 仅对分类 BCE 损失应用权重，边界框和分割损失保持不变

**实现类**：

- `WeightedSegmentationLoss`：继承 `v8SegmentationLoss`，重写分类损失计算
- `WeightedSegmentationModel`：继承 `SegmentationModel`，使用加权损失
- `WeightedSegmentationTrainer`：继承 `SegmentationTrainer`，集成加权模型

### 3.4 缺陷成因分析模型

对于检测到的缺陷区域，系统使用 SigLIP（`google/siglip-so400m-patch14-384`）进行成因语义分析。SigLIP 是 Google 发布的视觉语言模型，通过图文对比学习将图像和文本映射到同一语义空间。

选择 SigLIP 而非 CLIP 的原因：

1. SigLIP 使用 Sigmoid 损失替代 Softmax，在多标签场景下更稳定。
2. 在相同参数量下，SigLIP 的零样本分类精度优于 CLIP。
3. Hugging Face Transformers 库原生支持，集成成本低。

成因分析通过子进程隔离执行，避免 Windows 环境下 OpenCV 与 SigLIP 的线程冲突。主进程将裁剪区域编码为 base64，子进程加载 SigLIP 模型执行图文匹配，结果通过 JSON 返回。

### 3.5 缺陷成因 Prompt 设计

系统为 6 类缺陷设计了共 28 条成因描述（cause_prompts.json），每条包含英文特征描述、中文成因名称和排查建议。以裂缝为例：

| 成因         | 英文特征描述                                                           | 排查建议                                 |
| ------------ | ---------------------------------------------------------------------- | ---------------------------------------- |
| 塑性收缩     | irregular short fine mesh-like cracks on concrete surface              | 结合浇筑后早期养护和失水情况排查         |
| 干燥收缩     | parallel straight cracks or map-like cracking along the long direction | 复核养护条件、环境湿度和构件收缩约束情况 |
| 温度应力     | through crack perpendicular to temperature gradient                    | 结合温差、浇筑厚度和水化热控制记录排查   |
| 不均匀沉降   | diagonal crack with wider upper part and narrower lower part           | 复核基础沉降、支座变形和周边构件位移     |
| 钢筋锈蚀膨胀 | longitudinal crack along reinforcement direction with rust stain       | 检查保护层厚度、钢筋锈蚀和氯离子侵蚀情况 |
| 冻融循环     | map-like cracks with loose edges and surface scaling                   | 结合冻融环境、饱水状态和表面剥落范围排查 |

完整的 6 类 28 条 prompt 覆盖了裂缝、破损/剥落、蜂窝麻面、孔洞、钢筋外露、渗水的典型成因。

### 4.5 学习率调度

**线性衰减方案**（对照组和加权损失组）：学习率从 0.01 线性衰减至 0.0001，配合 3 个 epoch 的 warmup。

**余弦衰减方案**（最佳方案）：学习率按余弦曲线从 0.01 衰减至接近 0，在训练初期下降较慢、后期加速下降，有利于模型在早期充分探索参数空间。

## 5. 超参数调整策略

### 5.1 初始配置

训练使用 Ultralytics 的默认超参数作为起点，这些参数在 COCO 等大规模数据集上经过充分验证，具有较好的通用性。

### 5.2 Batch Size 选择

选择 batch_size=32，兼顾训练稳定性和显存占用。较大的 batch size 有助于 Mosaic 增强的效果发挥，同时使梯度估计更稳定。

### 5.3 图像尺寸选择

选择 imgsz=640，这是 YOLO 系列的标准输入尺寸。640×640 在保留足够细节的同时，推理速度可控。对于细小裂缝，更高的分辨率（如 1280）可能带来精度提升，但会显著增加训练和推理成本。

### 5.4 Mosaic 关闭策略

设置 `close_mosaic: 10`，在第 10 个 epoch 后关闭 Mosaic 增强。这样做的原因是：Mosaic 将 4 张图拼接，会改变目标的相对尺度，在训练后期可能干扰模型对真实场景的适应。关闭 Mosaic 后，模型在"干净"的原始图像上进行最后几轮微调。

### 5.5 类别加权策略

针对类别不平衡问题，实验组引入了类别加权损失。权重使用 sqrt 逆频率计算，上限 3.0：

| 类别          | 训练实例数 | 权重  |
| ------------- | ---------- | ----- |
| Crack         | 4,893      | 1.569 |
| Breakage      | 12,038     | 1.000 |
| Comb          | 4,356      | 1.662 |
| Hole          | 818        | 3.000 |
| Reinforcement | 9,504      | 1.125 |
| Seepage       | 3,746      | 1.793 |

可以看到 Breakage 实例数最多（12,038），权重最低（1.000）；Hole 实例数最少（818），权重达到上限（3.000）。

**实验结论**：

- 加权损失+余弦学习率方案在整体 mAP@25 上取得三组最优
- 最佳早停窗口为 28-30 个 epoch
- 余弦学习率在训练初期能更高效地引导模型逼近最优解

| YOLO26m-seg 官方 | COCO val2017 | 80 | ~118K | 52.5% |
| 本模型（部署） | crack-multi-seg test | 6 | 1,096 | **22.8%** |

差距来源：COCO 数据集规模远大于本项目且为人工标注，混凝土缺陷类别间视觉相似度高。

### 6.4 推理速度

| 环境 | GPU | 单帧耗时 | 首次加载 |
| ---- | --- | -------- | -------- |
| 本地推理 | NVIDIA RTX 4060 Laptop (6GB) | **~185ms** | 5-15s（权重加载到显存） |
| 训练环境 | NVIDIA RTX 4080 SUPER (32GB) | — | — |

预处理约 3ms，后处理约 1ms。GPU 显存加载模型权重约 5-15 秒，加载完成后后续帧推理恢复正常。

### 6.5 结果分析

当前最佳模型在测试集上的 mAP@25 为 0.457，处于中等水平。主要影响因素：

1. **SAM 生成的 mask 质量**：CUBIT-Det 和 GYU-DET 的分割标注由 SAM 自动生成，可能存在边缘不精确、过分割或欠分割问题，引入了标注噪声。
2. **类别不均衡**：6 类缺陷的样本数量差异较大，小样本类别检测精度偏低。加权损失+余弦学习率方案使少样本类获得提升（Comb +11.7%），但少数类仍受限于训练样本绝对不足。
3. **训练已收敛**：模型在 28-30 epoch 后已基本收敛，继续训练收益有限且存在过拟合风险。

### 7.2 核心模块

| 模块         | 文件                       | 职责                                                 |
| ------------ | -------------------------- | ---------------------------------------------------- |
| FastAPI 入口 | `app/main.py`            | 路由定义、会话管理、静态文件服务                     |
| 推理引擎     | `app/inference.py`       | YOLOv26-Seg 加载、推理、实例合并、特征提取、视频追踪 |
| 成因分析     | `app/cause_analyzer.py`  | SigLIP 子进程调度、prompt 管理                       |
| 成因工作器   | `app/cause_worker.py`    | SigLIP 模型加载与图文匹配（子进程）                  |

| 报告生成     | `app/reporter.py`        | Excel（openpyxl）和 PDF（reportlab）报告             |

### 7.3 检测流程

1. **图像输入**：用户上传图片或视频帧。
2. **YOLOv26-Seg 推理**：输出每个候选区域的 mask 多边形、边界框、置信度和类别。
3. **实例合并**：使用 Union-Find + IoU 阈值（0.3）将同一裂缝的多个分割结果合并。
4. **几何特征提取**：计算面积占比、细长度、主方向、边界复杂度、骨架长度、分支数量等。
5. **SigLIP 成因分析**：裁剪候选区域，与 28 条 prompt 进行图文匹配，输出各成因的相似度分数。
6. **可视化标注**：在原图上叠加半透明 mask、边界框和类别标签。
7. **报告导出**：生成包含缺陷类型、数量、位置坐标、置信度和成因分析的 Excel/PDF 报告。

### 7.4 视频追踪

视频检测采用逐帧推理 + BoT-SORT 目标追踪的方案：

- 用户设置采样间隔（默认每 5 帧取 1 帧）和最大帧数（默认 200 帧）。
- 每个追踪目标分配唯一 track_id，系统记录其在所有帧中的出现情况。
- 对每个追踪目标，选取置信度最高的帧作为"最佳代表帧"，生成裁剪图用于成因分析。
- 最终输出包含每个缺陷实例的时间线（首次/末次出现时间、最佳帧位置）。

### 7.5 WebSocket 实时检测

系统支持通过 WebSocket 进行实时帧检测，采用两阶段返回策略：

1. **第一阶段**（segmentation_done）：快速返回 YOLO + 实例合并结果，前端立即展示。
2. **第二阶段**（analysis_done）：返回 SigLIP 成因分析补充结果。

这种设计让用户先看到检测框，再看到成因分析，交互体验更流畅。

### 7.6 API 接口

| 接口 | 方法 | 说明 |
| ---- | ---- | ---- |
| `/api/detect/image` | POST | 单张图像检测，返回 JSON 结果 + base64 标注图 |
| `/api/detect/batch` | POST | 批量图像检测（最多 50 张），返回汇总统计 |
| `/api/detect/video` | POST | 视频检测与追踪，返回缺陷时间线 |
| `/ws/detect` | WebSocket | 分阶段检测，两阶段推送分割结果和成因分析 |

## 8. 总结

本项目完成了一套从数据处理、模型训练到应用系统集成的全流程基础设施缺陷检测系统。主要成果包括：

1. **数据集构建**：整合三个来源的数据集，通过 SAM 模型生成分割标注，构建了包含 12,935 张图像、6 类缺陷的 crack-multi-seg 数据集。
2. **模型训练**：使用 YOLOv26-Seg 在 crack-multi-seg 上进行三组对照实验：
   - 标准损失方案：mAP@25=0.450
   - 加权损失方案：mAP@25=0.452
   - **加权损失+余弦学习率方案：mAP@25=0.457（最优，选为部署模型）**
   - 加权损失+余弦学习率方案使少样本类（Comb +11.7%、Crack +2.4%）检测精度显著提升
3. **类别加权实验**：实现了自定义加权损失函数，实验证明与余弦学习率配合在 28-30 epochs 时效果最佳，继续训练会导致过拟合。
4. **系统开发**：基于 FastAPI 构建了支持图像/批量/视频三种输入模式的检测系统，集成 SigLIP 成因分析（28 条 prompt，6 类缺陷）、可视化标注和报告导出功能。
5. **技术创新**：采用分割模型 + 视觉语言模型的融合架构，兼顾缺陷定位精度和成因分析的可解释性。

当前系统的主要局限在于 SAM 生成的 mask 质量不均匀，以及类别不平衡对少数类检测精度的影响。后续工作可集中在数据质量优化、模型规模升级，以及在实际工程场景中的部署测试。
